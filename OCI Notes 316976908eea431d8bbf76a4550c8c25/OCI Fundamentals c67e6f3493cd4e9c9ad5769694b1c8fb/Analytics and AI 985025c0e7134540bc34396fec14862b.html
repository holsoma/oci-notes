Analytics and AI

  • contains mainly four services
    • data integration - native cloud service that helps with ETL
    • data flow - service for running apache spark applications and provides easy runtime environment to execute them
    • data catalog - metadata management service that helps data professionals discover data and support data governance topics which are really important for enterprises
    • data science platform - helps data scientists rapidly build, train and deploy machine learning models

Data Integration

  • oracle-managed ETL service (extraction, transformation, loading)
  • nice graphical interface to design data flows - data engineers with a deep understanding of data can develop their own data integration pipelines without technical knowledge
  • how does it work?
    • ingest data from sources (files, dbs, big data formats like AVRO and Parquet), register sources as data assets
    • design data integration flows (cleanse, transform and process the data), you can use built-in or construct your onw using visual data flow designer (drag and drop)
    • load data into autonomous database, MySQL, or data lake
  • advantages of service
    • helps in ingesting, cleansing, transforming, and reshaping data
    • helps in common ETL tasks
    • fully managed multi-tenant native cloud service

Data Flow

  • serverless fully managed OCI service to run apache spark applications at any scale with virtually zero administration
  • apache spark and what makes it unique
    • it is the unified analytics engine for big data processing with built-in modules for streaming, SQL, machine learning, and graph processing
    • it is unique, because it is the only software framework that combines data and AI
    • you can do large scale data transformations using spark then immediately run machine learning and AI algorithms on that data - useful for building large scale AI services
  • data flow analyzes data from OCI object storage and autonomous data warehouse, and even data form third-part RD systems
  • data flow concepts : user
    • applications
      • a spark application that uses the spark API to perform distributed data processing tasks
      • once a data flow app is created, anyone can reuse without worrying about complexities of deploying it, setting it up, running it
      • first thing a user is to create an apache spark application
    • library
      • central repository of data flow applications - anyone can browse, search and execute applications published to the library
    • runs
      • every time app is run or executed, a "run is created"
      • output, logs and statistics are automatically captured and securely stored
  • data flow concepts : administrative layer
    • allows you to set service limits and create administrators (with full control over what applications can be run and what executions can happen, what runs happen)
  • data flow concepts: infrastructure
    • elastic compute (high performance compute on demand, clusters started and stopped automatically)
    • elastic storage (scalable cost-effective cloud storage, transparent encryption, retention policies)
  • since it's a fully managed serverless offering with near zero administrative overhead, it means that data flow:
    • allocates your VMs, runs the jobs, securely captures all the output, then shits the cluster down when it's not running
  • data flow concepts: security
    • leverages OCI identity and access management service for authentication and authorization

Data Catalog

  • a service that gives a self-service data discovery and governance solution for your enterprise data
  • a single collaborative environment to manage technical, business, and operational metadata (you can collect, organize, find, access, understand, enrich, and activate this metadata)
  • how data catalog really works?
    • harvest technical data form a wide range of supported data sources (using private or public APIs)
    • create and manage a common enterprise vocabulary with a business glossary (a hierarchy of categories, subcategories and terms with detailed rich text descriptions)
    • enrich harvested metadata with annotations, then link to data entities and attributes (after you have done that, you can search, browse and explore catalog)
    • you can integrate the enterprise class capabilities of your data catalog with other apps

Data Science

  • fully managed and serverless platform for data scientists to manage machine learning models
  • provides a collaborative project-driven workspace that enables self-service serverless access to infrastructure for data science workloads
  • rich integration with serverless offering, data flow (manage apache spark service), autonomous data warehouse, OSS
  • provides data scientists with an API gateway

Data Science Components

  • projects (to collaborate)
    • centralize, organize and document data science assets, such as notebook sessions and models
  • data science notebook sessions
    • interactive coding environments for building and training models
    • comes with many pre-installed open source and oracle developed machine learning and data science packages
  • models
    • defines a mathematical representation of your data and business processes
    • store, track, share and manage models (you can load into notebooks for testing)
  • accelerated data science SDK
    • python library included as part of OCI data science service
    • has a friendly UI to cover all the steps involved in the lifecycle of machine learning models - make common data science tasks faster, easier, and less error prone
