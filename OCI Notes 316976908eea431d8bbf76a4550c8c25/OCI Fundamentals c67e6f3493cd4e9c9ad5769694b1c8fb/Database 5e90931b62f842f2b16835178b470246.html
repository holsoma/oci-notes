Database

  • OCI offers single-node DB systems on either bare metal or VMs, and 2-node real application cluster DB systems on virtual machines
  • VM DB system
    • fast provisioning
    • one-node virtual DB system
    • create DB with logical volume manager as storage management layer
    • alternative is called standard provisioning, provisioned with oracle automatic storage management (ASM)
  • bare metal DB system with fast performance
    • single bare metal server running oracle Linux with locally attached NVMe storage
    • if this node fails, you can just launch another system and restore database from current backups
  • 2-node real application cluster (RAC) DB system
    • each node is assigned to a different fault domain
    • provides high availability
  • exadata cloud service
    • leverage power of exadata in the cloud
    • oracle workloads have highest performance running on this exadata cloud service
    • massive scale with managed exadata infrastructure
  • autonomous DBs (shared and dedicated)
    • cloud database that uses machine learning to automate routine management tasks usually done by DBA (eg. database tuning, security, backups, updates)
    • supports two kinds of workloads
      • autonomous data warehouse (ADW)
      • autonomous transaction processing (ATP)
    • with exadata, you hear 3 terms all the time
      • self-driving
      • self-securing
      • self-repairing
  • autonomous shared DB
    • provision and manage only the autonomous DB
    • Oracle handles exadata infrastructure development and management
    • multi-tenant shared environment
  • autonomous dedicated DB
    • exclusive use of exadata hardware
    • single tenant dedicated to you
  • other options: open source databases
    • MySQL, fully managed OCI service, by MySQL team working within Oracle
    • NoSQL, on-demand throughput and storage based provisioning, supports JSON, table and key-value data types with flexible transaction guarantees and single digit millisecond latencies
    • autonomous JSON database, geared towards developing NoSQL style applications that use JSON documents

Autonomous Databases

  • all database and infrastructure management processes (eg. monitoring, tuning) are automated, and your DBS can focus on more important tasks
  • autonomous DBs have complete infrastructure automation, complete database automation and automated data center operations and machine learning (automates entire database stack, don't have to do repetitive and error prone tasks)
  • shared - you provision and manage only the autonomous database, oracle handles exadata infrastructure deployment and management
    • supported for both ATP and ADW
  • dedicated - exclusive use of the exadata hardware and underlying hardware, autonomous DB and exadata completely dedicated to you
    • supported for both ATP and ADW
  • autonomous transaction processing (ATP) for OLTP
  • autonomous data warehouse (ADW) for online analytical processing
  • complete database automation - self-securing, capabilities to protect against both external attack and malicious internal users
    • provisioning, scale-up and scale-out, tuning, security and patching, fault tolerance
    • RAC, data guard, database vault features still available
    • mission critical, simple, low risk and cost
  • does not support MySQL

DB Systems

  • fully managed oracle DBs running in cloud
  • support enterprise or standard edition
    • 4-tiers of oracle licensing options, including BYOL
  • can be run in VMs or bare metal shapes
  • exadata is supported
  • available in OCI regions around the world
  • cloud automation under customer control - provisioning, patching, backup, disaster recovery
  • oracle DB on VMs
    • up to 24 OCPUs
    • up to 320 GB ram
    • up to 40TB of usable block-volume storage
    • remember: data in block storage can be stored beyond lifecycle of the compute instance itself
  • oracle RAC on VMs
    • up to 48 OCPUs
    • up to 640GB ram
    • up to 40TB of usable block-volume storage
    • high availability by creating two nodes in RAC configuration
    • extreme performance edition
  • oracle DB on bare metal
    • up to 52 OCPUs
    • 768GB ram
    • up to 16TB of usable NVMe local storage
  • quickly provision something -> use VM
  • fast performance -> use bare metal option
  • even greater performance -> use exadata DB systems
  • high availability is supported
    • using real application cluster (RAC)
    • data guard, where you can create a primary and a stand by, synchronizing your data in the same region or across regions
  • scalability
    • through dynamic CPU and storage scaling
    • increase up and down and change your storage
  • security
    • inbuilt in multiple layers, security in the infrastructure layer through thing like identity, access management, authZ, authN
    • there is security within the database, transparent data encryption, encrypted backups, encrypted block volumes
  • you can BYOL, permanent billing is supported

MySQL

  • host mySQL databases in the cloud
    • rapid provisioning of MySQL server instances
    • easy to test, apply and roll back patches
  • scale quickly, cheaply and efficiently
    • elastic compute - increase or decrease resources as required
    • elastic storage - expand storage capacity when your data grows
    • elastic replication - distribute workloads dynamically
  • improve security by protecting against attacks and misuse with advanced tools, ensure regulatory compliance
  • reduces cost of ownership because you are now using MySQL as a cloud service instead of running it on your own
  • high availability- fault tolerant, automatic failover, zero data loss (create high availability database rather than standalone MySQL database)
  • standalone
    • standalone is good for dev and test environment
    • back-ended by resilient and secure OCI block volume
  • high availability
    • high availability option is good for production environment and enables application to meet higher uptime requirements
    • set up the zero data loss tolerance, in the case of a failure, there is automatic failover within minutes toa secondary instance + no reconfiguration needed
  • MySQL database service with heatwave
    • heatwave is a new integrated, high-performance, in-memory query accelerator
    • by order of magnitude for analytics and transaction queries
    • scales out to thousands of cores
    • only service that allows DBAs to run both OLTP and OLAP workloads directly from MySQL database

NoSQL

  • fully managed
    • database operation, maintenance tuning managed by oracle
  • elastic
    • dynamically change throughput and storage capacities based on workloads
  • high performance
    • predictable single-digit millisecond latencies with high workloads
  • data model flexibility
    • document, fixed schema key/value models supported with a single application interface
  • access control
    • enterprise grade security with roles and privileges
  • low operating cost
    • pay only for the throughput and storage capacities provisioned
  • developer friendly
    • easy to use APIs and integrated with different developer tools
  • always available
    • built-in high availability to ensure business continuity
  • hybrid cloud
    • NoSQL with on premise solution using a single application interface
