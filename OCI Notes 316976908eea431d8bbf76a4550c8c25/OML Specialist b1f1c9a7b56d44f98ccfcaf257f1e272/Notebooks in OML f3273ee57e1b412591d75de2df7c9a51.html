Notebooks

  • web-based interface to
    • access algorithms in your ML library
    • perform data analytics
    • code ML solutions
    • collaborate with other analytics developers
    • create visualizations
    • add text and equations to support code by providing context
  • based on Apache Zeppelin technology
  • create a name for your notebook, then add connection groups in the connections field (global connection group is default)
  • in edit mode, you can:
    • write code to fetch data
    • run paragraphs
    • perform data discovery and visualization
  • use dynamic forms to provide an easy way to filter the data (text, select, check box)

Text Input Forms

  • dynamically retrieve values that are defined
  • to create a text input form, use $ {formName}
  • SQL statement presents user with text field where the user can enter the table to view all the table objects

Select Forms

  • select an input value from a list, then dynamically retrieve corresponding value from data set
  • to create a select form, use $ {formName=defaultValue, option1 | option2}
  • SQL statement selects all objects where object type is either INDEX, TABLE, VIEW, or SYNONYM
  • SQL statement presents user with dropdown list containing specified options
    • selecting one of the options will display corresponding objects in output

Check Box Forms

  • provide multiple selections of inputs as check box options (users can choose more than one)
  • to create a check box form, use $ {checkbox:formName=defaultValue1|defaultValue2, option1|option2}
  • SQL statement checks OWNER, OBJECT_TYPE and STATUS columns of all objects
  • SQL statement displays columns of the ALL_OBJECTS table for which the corresponding check box is selected

Versioning and Setting up an Output Format

  • OML provides versioning, you can
    • revert to an older version
    • delete a particular version
    • create a new notebook from a particular version
  • you can display your query output in a particular format in OML
    • use the set SQL format command to specify output format
    • SET SQLFORMAT format_option, subsequent SELECT commands will return output in specified format (this formatting is available in script interpreter, NOT SQL intepreter, hence you need %script)
  • there are other formats available, like CSV, HTML and JSON
    • the content of the table can change dynamically to match the search string
    • FIXED format, SET SQLFORMAT FIXED
      • produces a report with fixed width columns for all the data enclosed in double quotes
    • DEFAULT option
      • clear previous custom SQL format settings and reverts to default original output format
    • LOADER option
      • produce an output that is type character delimited with string values enclosed in double quotes
