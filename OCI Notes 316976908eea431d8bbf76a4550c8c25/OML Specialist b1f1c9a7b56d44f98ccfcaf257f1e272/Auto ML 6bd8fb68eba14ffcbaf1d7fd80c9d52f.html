Introduction to Oracle AutoML

  • autoML automates the machine learning workflow
  • can be implemented through OML for Python and AutoML UI
  • the pipeline consists of four steps (algorithm selection, adaptive sampling, feature selection, and hyperparameter tuning)
    • algorithm selection - identifies top k algorithms that maximizes model score
    • adaptive sampling - finds the smallest sample size that represents the full data set
    • feature selection - subset of features that are highly predictive of the target are selected
    • hyperparameter tuning - optimal values for the model’s hyperparameters are found

Algorithm Selection

  • choose an algorithm that provides a model with the highest predictive accuracy (processes a list of algorithms and ranks them according to performance)
  • steps involved:
    • extract characteristics in dataset like shape and feature correlations
    • invoke specialized score prediction models, capable of predicting algorithm performance across a variety of datasets and domains
    • rank algorithms based on predicted performance, select optimal algorithm (smaller the dataset the better the model performance → processing time is lesser [real-world datasets are significantly large!])
    • select top k algorithms

Adaptive Sampling

  • select a sample of a data set which adequately represents the original data set (make the data set smaller → shorten processing time)
  • iteratively sub-samples the dataset and evaluates each sample to obtain a model score → used to find smallest sample size that represents data set
  • steps involved:
    • identify representative sample for given data set and algorithm
    • predict algorithm performance on the sample (measure model score), optimize until score converges

Feature Selection

  • choose features in dataset which are highly predictive of the target
  • only select those features to increase speed of model training
  • accelerated data science (ADS) to identify optimal feature subset
  • steps involved:
    • obtain meta features of the dataset (recommended features by dataset’s metadata/past dataset characterizations)
    • use multiple ranking algorithms to rank the features
      • for each ranking, find optimal feature subset
      • predict algorithm performance on a given feature subset
    • iterate over multiple ranking algorithms

Hyperparameter Tuning

  • determine optimal value for model’s hyperparameters
  • usually time-consuming
  • focuses on improving chosen algorithm score on reduced data set (given by adaptive sampling and feature selection)
  • ADS novel algorithm to search across hyperparameter dimensions, automatic convergence when optimal hyperparameters are defined and identified

AutoML with OML4Py

  • oracle advanced meta-learning intelligence to learn from a repertoire of datasets with the goal of avoiding exhaustive searches, reducing compute time and cost

oml.automl.AlgorithmSelection

  • oml.automl.AlgorithmSelection class (supports both classification and regression problems)

oml.automl.FeatureSelection

  • usues several feature ranking algorithms to identify best feature subset, without comprising model performance

oml.automl.ModelTuning

  • tunes hyperparameters
  • automates time-consuming process using highly-parallel, asynchronous, gradient-based hyperparameter optimization algorithm

oml.automl.ModelSelection

  • class to automatically select OML algorithm according to selected score metric and tune that algorithm

OML AutoML UI

  • provides a no-code UI to automate machine learning process
  • you can select the model you want to deploy through REST endpoint or immediately through SQL queries or OAC or APEX
  • experiments that follow a pipeline which consists of the following activities
  • monitored with minimal user input required in the entire lifecycle of the experiment
  • when managing models, you can deploy existing OML in-database model as a REST endpoint, un-deploy, and change namespace of a previous deployed model
  • results shown in a leaderboard according to model metric selected → you can create notebook using OML4Py that will recreate selected AutoML UI model with same settings
  • opt for “faster results” if you want to get candidate models sooner at the expense of accuracy
  • recall: proportion of actual positives that are correctly classified
  • precision: proportion of predictive positives that are true positives
  • F1 score: combines precision and recall into a single number, computed using harmonic mean

Deploying

  • unique alphanumeric name with max 50 characters
  • alphanumeric URI with max 200 characters
  • version number XX.XX (2.0, 3.1)
  • successful deployed model will be listed in the deployments page, you can view:
    • model metadata (model name, mining function, algorithm, attributes, etc)
    • REST API URI
  • you can choose deployed model and create notebook using the model
