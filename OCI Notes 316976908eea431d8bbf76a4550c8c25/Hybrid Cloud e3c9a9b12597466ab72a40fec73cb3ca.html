Hybrid Cloud

Hybrid Cloud

  • deployment size (small or large) and deployment model (self-contained or remotely tethered)
    • self-contained model
      • customers get the same billing model, operational model, security, and same set of services
    • remotely tethered
      • OCI exadata cloud@customer (hybrid cloud offering)
      • the control plane or the service management runs in the public cloud to take advantage of existing services, governance, and operational control (hardware and data remain on prem)
      • when you log onto the console, you can manage your cloud resources, as well as your on-premises resources - exadata cloud@customer (high performance, directly to a customer's data center)
      • why use remotely tethered?
        • ensure strict data sovereignty (data subject to laws of the country it is located in) and security by retaining all the data on prem
        • shift operational management to an OCI managed cloud service
        • pretty big scale, not the same scale as a public cloud, but decent scale and performance
    • roving edge infrastructure (self-contained model)
      • small-size deployment
      • it is a service that extends fundamental OCI capabilities to a ruggedized device (hardware designed to have increased durability to tolerate harsh conditions - extreme temps, shocks, drops, water, dust)
      • key component is the roving edge device (RED)
        • high-powered and portable server node
        • customers configure a RED in their OCI console with the same services, applications, and data they use in the cloud
        • devices are shipped to the customer
          • for eg. you can use these devices in a remote location without internet connectivity, then take them to a transfer site once in a while to do a data-sync
          • this device can do a certain subset of what you can do in public cloud in your own environment, even without internet connectivity

    • oracle cloud VMware solution (OCVS)
      • VMware is the foundation of majority of enterprise data centers
      • to date, enterprises have chosen to deploy VMware with full control on-premises or delegate control to a managed service provider/hosting provider
      • in the past few years, customer started deploying their VMware workloads to different public clouds
        • this is because public clouds offer benefits like just-in-time provisioning, elastic capacity scaling, and pay-for usage models (benefits not found in traditional data centers)
        • however, running VMware in other public clouds require you to relinquish full control over administration, access and upgrades (other than oracle cloud!)
        • oracle cloud VMware is the ONLY solution that provides a native VMware based cloud environment with complete control using familiar VMware tools
          • varying levels of control - low to high, across several dimensions
          • customers can choose when and whether they want to upgrade their clusters (complete control over upgrade, patching)
          • admin access is also a feature - they own root credentials and metadata for their cluster, and they can make changes
          • they have been managing VMware on prem themselves, they can do the same using this service to move or extend on prem workloads to OCI across all regions (including on prem dedicated regions without having to re-architect or retool any of the operations)

  • hybrid cloud computing allows for security, consistent management, and control

Dedicated Regions

  • secure, 99.95% availability, meets data residency and latency requirements, self-contained cloud region
  • fully managed cloud region that brings all of our cloud services into customer data centers (in a self-contained model)
  • what does that mean?
    • we put the exact same software (control plane, data plane - as in the public cloud) and hardware into your data centers
  • why would we do this?
    • for data sovereignty and compliance
    • many industries have strong regulations, so you want to keep data on prem (best of both worlds) - you get operational scalability, manageability, security
    • this is completely symmetric, no other cloud vendor as this kind of capability existing today
      • more than 80 services available in the public cloud, and are available in the dedicated region as well
      • same operation model, same SLAs
      • same economics - pay as you go model, same support model

Oracle Cloud VMware solution

  • just in time provisioning, elastic capacity, ability to scale up and down, pay for usage model
  • traditional cloud services require giving up full control over administration, access and upgrades
  • OCI offers all these benefits while still giving VMware administrators full control over their environment (we are running VMware on OCI on top of bare metal compute instances, but you have all VMware software running on it)
    • integrated solution (VMware software + OCI infrastructure like bare metal compute [that you can expand at will]
    • adv: you can upgrade and patch your clusters, log in as root users, and don’t need to follow what the cloud provider is doing
  • you don’t feel a difference between on prem and cloud, you can leverage the same experience, tools and processes that you are running on prem
